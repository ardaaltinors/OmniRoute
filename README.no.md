## рҹҺҜ Hva OmniRoute lГёser вҖ” 30 ekte smertepoeng og brukstilfeller

> **Hver utviklere som bruker AI-verktГёy mГёter disse problemene daglig.** OmniRoute ble bygget for ГҘ lГёse dem alle вҖ“ fra kostnadsoverskridelser til regionale blokker, fra Гёdelagte OAuth-flyter til protokolloperasjoner og observerbarhet i bedrifter.

<details>
<summary><b>рҹ’ё 1. "Jeg betaler for et dyrt abonnement, men blir fortsatt avbrutt av grenser" </b></summary>

Utviklere betaler $20вҖ“200/mГҘned for Claude Pro, Codex Pro eller GitHub Copilot. Selv om du betaler, har kvoten et tak вҖ“ 5 timers bruk, ukentlige grenser eller rategrenser per minutt. MidtkodingsГёkt, leverandГёren slutter ГҘ svare og utvikleren mister flyt og produktivitet.

**Hvordan OmniRoute lГёser det:**

- **Smart 4-lags fallback** вҖ” Hvis abonnementskvoten gГҘr tom, omdirigeres automatisk til API-nГёkkel вҶ’ Billig вҶ’ Gratis med null manuell intervensjon
- **Sanntidskvotesporing** вҖ” Viser tokenforbruk i sanntid med tilbakestilt nedtelling (5 timer, daglig, ukentlig)
- **StГёtte for flere kontoer** - Flere kontoer per leverandГёr med automatisk round-robin - nГҘr en gГҘr tom, bytter du til den neste
- **Egendefinerte kombinasjoner** вҖ” Tilpassbare reservekjeder med 6 balansestrategier (fyll fГёrst, round-robin, P2C, tilfeldig, minst brukt, kostnadsoptimalisert)
- **Codex Business Quotas** вҖ” OvervГҘking av bedrifts-/teamarbeidsomrГҘdekvoter direkte i dashbordet

</details>

<details>
<summary><b>рҹ”Ң 2. "Jeg trenger ГҘ bruke flere leverandГёrer, men hver av dem har en annen API" </b></summary>

OpenAI bruker ett format, Claude (Anthropic) bruker et annet, Gemini enda et annet. Hvis en utvikler Гёnsker ГҘ teste modeller fra forskjellige leverandГёrer eller fallback mellom dem, mГҘ de rekonfigurere SDK-er, endre endepunkter, hГҘndtere inkompatible formater. Tilpassede leverandГёrer (FriendLI, NIM) har ikke-standardmodellende endepunkter.

**Hvordan OmniRoute lГёser det:**

- **Unified Endpoint** - En enkelt `http://localhost:20128/v1` fungerer som proxy for alle 36+ leverandГёrer
- **Formatoversettelse** вҖ” Automatisk og gjennomsiktig: OpenAI вҶ” Claude вҶ” Gemini вҶ” Responses API
- **Responsrensing** вҖ” Fjerner ikke-standardfelter (`x_groq`, `usage_breakdown`, `service_tier`) som bryter OpenAI SDK v1.83+
- **Rollenormalisering** вҖ” Konverterer `developer` вҶ’ `system` for ikke-OpenAI-leverandГёrer; `system` вҶ’ `user` for GLM/ERNIE
- **Think Tag Extraction** вҖ” Trekker ut `<think>`-blokker fra modeller som DeepSeek R1 til standardiserte `reasoning_content`
- **Structured Output for Gemini** вҖ” `json_schema` вҶ’ `responseMimeType`/`responseSchema` automatisk konvertering
- **`stream` er standard til `false`** вҖ” Justerer med OpenAI-spesifikasjoner, og unngГҘr uventet SSE i Python/Rust/Go SDK-er

</details>

<details>
<summary><b>рҹҢҗ 3. "Min AI-leverandГёr blokkerer min region/land" </b></summary>

LeverandГёrer som OpenAI/Codex blokkerer tilgang fra visse geografiske omrГҘder. Brukere fГҘr feil som `unsupported_country_region_territory` under OAuth- og API-tilkoblinger. Dette er spesielt frustrerende for utviklere fra utviklingsland.

**Hvordan OmniRoute lГёser det:**

- **3-Level Proxy Config** вҖ” Konfigurerbar proxy pГҘ 3 nivГҘer: global (all trafikk), per leverandГёr (kun Г©n leverandГёr) og per tilkobling/nГёkkel
- **Fargekodede proxy-merker** вҖ” Visuelle indikatorer: рҹҹў global proxy, рҹҹЎ leverandГёrproxy, рҹ”ө tilkoblings proxy, viser alltid IP
- **OAuth-tokenutveksling gjennom proxy** - OAuth-flyt gГҘr ogsГҘ gjennom proxyen, og lГёser `unsupported_country_region_territory`
- **Test av tilkobling via proxy** вҖ” Tilkoblingstester bruker den konfigurerte proxyen (ikke mer direkte forbikobling)
- **SOCKS5-stГёtte** вҖ” Full SOCKS5-proxystГёtte for utgГҘende ruting
- **TLS-fingeravtrykkspoofing** вҖ” Nettleserlignende TLS-fingeravtrykk via `wreq-js` for ГҘ omgГҘ botdeteksjon

</details>

<details>
<summary><b>рҹҶ“ 4. "Jeg vil bruke AI for koding, men jeg har ingen penger" </b></summary>

Ikke alle kan betale $20вҖ“200 per mГҘned for AI-abonnementer. Studenter, utviklere fra fremvoksende land, hobbyfolk og frilansere trenger tilgang til kvalitetsmodeller uten kostnad.

<details>
<summary><b>рҹ”’ 5. "Jeg trenger ГҘ beskytte AI-gatewayen min mot uautorisert tilgang" </b></summary>

NГҘr du eksponerer en AI-gateway til nettverket (LAN, VPS, Docker), kan alle med adressen konsumere utviklerens tokens/kvote. Uten beskyttelse er API-er sГҘrbare for misbruk, umiddelbar injeksjon og misbruk.

**Hvordan OmniRoute lГёser det:**

- **API Key Management** вҖ” Generering, rotasjon og scoping per leverandГёr med en dedikert `/dashboard/api-manager`-side
- **Tillatelser pГҘ modellnivГҘ** вҖ” Begrens API-nГёkler til spesifikke modeller (`openai/*`, jokertegnmГёnstre), med Tillat alt/begrens
- **API Endpoint Protection** вҖ” Krev en nГёkkel for `/v1/models` og blokker spesifikke leverandГёrer fra oppfГёringen
- **Auth Guard + CSRF Protection** вҖ” Alle dashbordruter beskyttet med `withAuth` mellomvare + CSRF-tokens
- **Rate Limiter** вҖ” Per-IP ratebegrensning med konfigurerbare vinduer
- **IP-filtrering** вҖ” Tillatelsesliste/blokkeringsliste for tilgangskontroll
- **Prompt Injection Guard** вҖ” Sanitisering mot ondsinnede spГёrsmГҘlsmГёnstre
- **AES-256-GCM-kryptering** вҖ” Legitimasjon kryptert i hvile

</details>

<details>
<summary><b>рҹӣ‘ 6. В«Tilbyderen min gikk ned og jeg mistet kodeflyten minВ» </b></summary>

AI-leverandГёrer kan bli ustabile, returnere 5xx-feil eller nГҘ midlertidige hastighetsgrenser. Hvis en utvikler er avhengig av en enkelt leverandГёr, blir de avbrutt. Uten strГёmbrytere kan gjentatte forsГёk krasje applikasjonen.

**Hvordan OmniRoute lГёser det:**

- **Circuit Breaker per leverandГёr** вҖ” Automatisk ГҘpning/lukking med konfigurerbare terskler og nedkjГёling (Lukket/Г…pen/HalvГ…pen)
- **Eksponentiell backoff** вҖ” Progressive forsinkelser pГҘ nytt forsГёk
- **Anti-tordenflokk** вҖ” Mutex + semaforbeskyttelse mot samtidige stormer pГҘ nytt forsГёk
- **Combo Fallback Chains** вҖ” Hvis primГҰrleverandГёren mislykkes, faller den automatisk gjennom kjeden uten inngrep

- **Helsedashbord** вҖ” OppetidsovervГҘking, strГёmbrytertilstander, sperringer, cachestatistikk, p50/p95/p99 latency

<a name="oauth-em-servidor-remoto"></a>

> **вҡ пёҸ VIKTIG for bruk av OmniRoute med VPS/Docker/server-fjernkontroll**

### OAuth

Os testedores **Antigravity** og **Gemini CLI** usam **Google OAuth 2.0** for autenticaГ§ГЈo. O Google exige que a `redirect_uri` usada no fluxo OAuth seja **exatamente** uma das URIs pre-cadastradas no Google Cloud Console do aplicativo.

Som credenciais OAuth embutidas no OmniRoute estГЈo cadastradas **apenas para `localhost`**. Quando vocГӘ acessa o OmniRoute em um servidor remoto (eks: `https://omniroute.meuservidor.com`), o Google avviser autenticaГ§ГЈo com:

```

```

### LГёsning: Konfigurer OAuth-tilbehГёr

VocГӘ precisa criar um **OAuth 2.0 Client ID** no Google Cloud Console com en URI for seu service.

#### Passo a passo

**1. Tilgang til Google Cloud Console**

Abra: [https://console.cloud.google.com/apis/credentials](https://console.cloud.google.com/apis/credentials)

**2. Crie um novo OAuth 2.0 Client ID**

- Klikk dem **"+ Opprett legitimasjon"** вҶ’ **"OAuth-klient-ID"**
- Tipo de aplicativo: **"Nettapplikasjon"**
- Navn: escolha qualquer nome (eks.: `OmniRoute Remote`)

**3. Adicione som autorisert omdirigerings-URI**

Ingen campo **"Authorized redirect URIs"**, adicione:

```
https://seu-servidor.com/callback
```

> Substitua `seu-servidor.com` pelo domГӯnio ou IP do seu servidor (inclua a porta se necessГЎrio, ex: `http://45.33.32.156:20128/callback`).

**4. Salve e copy as credenciais**

ApГіs criar, o Google mostrarГЎ o **Client ID** e o **Client Secret**.

**5. Konfigurer som variГЎveis de ambiente**

No seu `.env` (ou nas variГЎveis de ambiente do Docker):

```bash
# Para Antigravity:
ANTIGRAVITY_OAUTH_CLIENT_ID=seu-client-id.apps.googleusercontent.com
ANTIGRAVITY_OAUTH_CLIENT_SECRET=GOCSPX-seu-secret

# Para Gemini CLI:
GEMINI_OAUTH_CLIENT_ID=seu-client-id.apps.googleusercontent.com
GEMINI_OAUTH_CLIENT_SECRET=GOCSPX-seu-secret
GEMINI_CLI_OAUTH_CLIENT_SECRET=GOCSPX-seu-secret
```

**6. Reinicie o OmniRoute**

```bash
# Se usando npm:
npm run dev

# Se usando Docker:
docker restart omniroute
```

**7. Tente conectar novamente**

Dashboard вҶ’ LeverandГёrer вҶ’ Antigravity (ou Gemini CLI) вҶ’ OAuth

Agora o Google redirecionarГЎ corretamente para `https://seu-servidor.com/callback` e autenticaГ§ГЈo funcionarГЎ.
